Memory DNA — Every Memory Has 8 Properties

Memory DNA: Rich, Structured, Living Memories
Unlike flat vector stores, every Solomon memory carries a DNA-like profile: Importance (0–1, how significant was this?), Emotional Valence (−1 to +1, positive or negative?), Confidence (0–1, how certain am I?), Connectivity (how many related memories link here?), Age (decay factor), Cognitive State (what was your mental state when this formed?), Access Frequency (how often retrieved?), Contradiction Score (how much does this conflict with other memories?). This DNA drives retrieval priority, decay rate, and consolidation decisions in the Dream Engine.

The Forgetting Engine

Solomon uses an exponential decay function modeled after the Ebbinghaus forgetting curve. Every memory has a Memory Reputation Score computed as:

Rₘ(t) = [α·Vₛ + β·ln(1+Uᵤ)] · e−λt · (1 − Cᵣ)

Memory Reputation Score · Rₘ ≤ 0.25 → pruned by Dream Engine

Where Vₛ is the epistemic verification signature, Uᵤ is access frequency (log-damped), λ is the horizon-specific decay constant, and Cᵣ is contradiction frequency. But crucially: the pattern of decay itself is information. A memory accessed frequently then suddenly abandoned signals an interest shift — Solomon records this as a behavioral event in L3.

Semantic Time — The New Innovation

Semantic Time: Memories Connected by When They Were Formed
Standard vector stores connect memories by content similarity. Semantic Time connects memories by the cognitive context in which they were formed. A memory created during "deep focus + excitement + 4pm + coding session" is indexed into a cognitive-context cluster alongside other memories formed under the same state — even if those memories have completely different content. This means Solomon can surface patterns across your life that content-based search would never find: "Every time you've been in this exact emotional-cognitive state, you've made this type of decision."

The Consciousness Continuity Protocol (CCP)

Zero-Reset Presence: Sessions Never Start From Zero
The biggest unsolved problem in AI is the reset. Every session, every device switch, every restart — you start over. Solomon solves this with the Consciousness Continuity Protocol. Before every graceful shutdown, Solomon serializes its complete cognitive state: active working memory, current emotional fingerprint, active goals with their progress vectors, the cognitive state at the moment of shutdown, and the priority queue of pending thoughts. On restart, this entire state is restored — Solomon picks up exactly where it left off, as if no time passed. On a new device, the same state transfers via encrypted sync. The relationship has no restart button.
