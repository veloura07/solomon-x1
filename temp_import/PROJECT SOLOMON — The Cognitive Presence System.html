Memory DNA

Every Memory Carries 8 Properties
Unlike flat vector stores, every Solomon memory carries a DNA-like profile: Importance (0–1), Emotional Valence (−1 to +1), Confidence (0–1), Connectivity (linked memory count), Age (decay factor), Cognitive State (your mental state when formed), Access Frequency (retrieval rate), Contradiction Score (conflict with other memories). This DNA drives retrieval priority, decay rate, and Dream Engine consolidation.

The Forgetting Engine

Rₘ(t) = [α·Vₛ + β·ln(1+Uᵤ)] · e−λt · (1 − Cᵣ)

Memory Reputation Score · Rₘ ≤ 0.25 → pruned by Dream Engine
Semantic Time: Memories Connected by When They Were Formed
Standard vector stores connect memories by content similarity. Semantic Time connects memories by the cognitive context in which they were formed. A memory created during "deep focus + excitement + 4pm + coding session" is indexed alongside other memories from that same state — even if the content is completely different. This surfaces patterns no content-based search would ever find.

22 — Simulation Brain & Decision Science
1000 Futures
Before Every Decision

The Simulation Brain (Brain 09) is the most computationally intensive module in Solomon. It uses Monte-Carlo Tree Search over a probabilistic model of your specific world to generate structured future scenarios — not generic advice, but predictions calibrated to your actual history, skills, biases, and current context.

Personalized Future Modeling — Not Generic Advice
When you ask "Should I start this startup now vs. in 6 months?", the Simulation Brain doesn't give you internet wisdom. It builds a decision tree where each node represents a choice or event, and each edge is weighted by probability estimated from: your historical decision outcomes (L9 Legacy Ledger), your current resource state (skills, time, network, capital), domain knowledge (market data from opportunity scanner), your known cognitive biases (extracted from Decision Archaeology), and your stated values (from Ideal-Self model). It runs 1000 MCTS simulations and returns 4 structured scenario types with probability estimates, key risk factors, and warning signs to watch.

4 Scenario Types

CONSERVATIVE — p=~0.6

What most likely happens given current trajectory. Risk factors identified. Actionable hedges suggested.

OPTIMISTIC — p=~0.2

Best-case path and what conditions need to be true for it to materialize. What actions maximize this path?

CONTRARIAN — p=~0.1

The non-obvious path most people miss. Cross-domain analogies from Decision Archaeology. "In your past, this type of situation resolved via unexpected channel."

CATASTROPHIC — p=~0.1

Downside case and its early warning signs. What are the tripwires? What do you do if you see them?

Cognitive Bias Detection

The Simulation Brain maintains a personal bias profile extracted from Decision Archaeology — the biases that specifically affect your decisions, not a generic list. Detected biases are applied as correction weights in the MCTS simulation:

Common Bias Patterns
  • Optimism bias in time estimates

  • Overweighting recent setbacks
  • Underweighting weak network signals

How Correction Works
  • Each detected bias gets a personal magnitude score (0–1)

Data Flow Privacy Guarantees

🟢 What Never Leaves Your Machine

  • Raw audio and video frames (processed in-process, immediately discarded)
  • Keystroke dynamics and mouse coordinates (only aggregated statistics stored)
  • Screen content (only extracted text/semantic content stored, not images)
  • Personal documents (indexed locally, embeddings stay local)
  • Relationship profiles and communication history
  • Decision archaeology and Legacy Ledger entries
  • Cognitive genome and behavioral baseline

🟡 What Can Optionally Sync (E2E Encrypted)

  • Memory summaries (never raw content) — for multi-device access
  • Cognitive genome settings — for consistent personality across devices
  • Quest progress and XP state — if user opts into cloud save
  • Anonymous CSI analytics — aggregate only, fully anonymized, opt-in research

🔴 What Never Syncs, Period

  • Raw behavioral signals (audio, video, keystrokes)
  • Decryption keys — derived at unlock, never stored remotely
  • Biometric data of any kind
  • Contents of any document processed by Solomon
  • Relationship profiles or personal communication data

Sensitive Content Classifier

Before any content from your screen or voice reaches Solomon's memory or analysis pipeline, a fast local DistilBERT classifier (11M params, <5ms inference) checks for sensitive categories and applies automatic redaction. No manual exclusion zones needed for common sensitive content types:

Auto-Excluded Categories
  • Banking and financial data (account numbers, card details, statements)
  • Passwords, API keys, credentials in any format
  • Medical records and health information (HIPAA-equivalent protection)
  • Personal identification (Aadhaar, PAN, passport numbers)
  • Private communications explicitly marked as personal
User-Definable Exclusions
  • Application exclusion list: specific apps whose content Solomon never sees
  • Time exclusion: "Privacy Mode" hours where no perception runs
  • Screen region exclusion: specific areas of display always blurred from capture
  • Contact exclusion: specific people whose communications are never processed
  • Topic exclusion: keyword-based redaction for sensitive personal topics
